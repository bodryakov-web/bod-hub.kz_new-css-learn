#Техническое задание на создание проекта NewCSSLearn.

##Задача.
Создай full-stack web-приложение представляющее собой сайт для обучения студентов новейшим свойствам CSS. Сам учебный курс в текстовом виде уже есть.

##Применяемый стек технологий.
Web приложение должно быть создано без использования backend или frontend фреймворков или библиотек. 
Backend должен быть организован на чистом PHP без фреймворков и библиотек. Тексты уроков и другая информация должна хранится в базе данных MySQL в виде JSON.
Frontend должен быть только чистый HTML, CSS, JavaScript. Обязательно без фреймворков и библиотек, в т.ч. React и Next. Чтобы можно было сделать деплой на простом хостинге.
Редактирование содержимого уроков должно осуществляться в редакторе Quill, подключаемого через CDN.

##Общее содержание и функционал сайта
Сайт должен выглядеть как последовательность уроков разбитых на разделы. Разделы могут меняться, добавляться и удаляться. Внутри разделов должны быть уроки. Уроки могут меняться, добавляться и удаляться.  Для образца сделай в 2 раздела по 2 урока в каждом. 
Будет примерно 10 разделов по 5-10 уроков в каждом разделе.
Для каждого раздела и для каждого урока, для хранения в базе данных должен быть slug, который может содержать только маленькие английские буквы и дефис.
Структура URL должна быть примерно такая: 
section_number-<section_name>/lesson_number-<lesson_name>
Отдельной Главной страницы не нужно. 
При заходе на сайт пользователь должен сразу попадать на страницу разделов. Разделы должны быть оформлены визуально как карточки. Каждая карточка раздела должна содержать заголовок раздела и кликабельные блоки-кнопки-ссылки заголовков уроков. Каждая такая кнопка-ссылка должна вести на страницу соответствующего урока.
Статистика по пользователям (кто какие уроки прошел) не нужна.
Каждый урок должен соответствовать отдельной странице.
HTML-страницы каждого урока должны формироваться динамически, при клике на заголовок или гиперссылку соответствующего урока соединяя единый шаблон урока и данные (учебный материал) полученный в формате JSON из базы данных.
Каждая страница должна быть в кодировке UTF-8 без BOM для корректного отображения текста на русском языке.
Нужно предусмотреть отдельную страницу 404/ошибки с кнопкой перехода на главную страницу.

##Описание урока
Каждый урок должен состоять из трёх секций:
1. теоретический материал, 
2. тестирование, 
3. задачи. 
Для секции теоретического материала заголовок "Теория" не нужен.
Тесты должны быть простого типа: вопрос, 4 варианта ответа из которых только один правильный. Количество тестовых вопросов может быть разным для каждого урока (примерно 5-10). Для образца сделай три тестовых вопроса в 1 уроке 1 раздела.
После каждого ответа - сразу, без нажатия кнопки проверить, пользователю показывается результат. Правильный ответ должен подсвечиваться зелёной галочкой, а неправильный - красным крестиком.
Прогресс и результаты прохождение тестов сохранять не нужно.
Условия задач выводятся просто как блоки текста.
Для задач интерактивный редактор с проверкой кода не нужен. Должно быть только условие задачи
В конце каждого урока должны быть кнопки предыдущий/следующий/в оглавление. В последнем уроке раздела, должна быть кнопка перехода на главную страницу.
Все данные урока, включая используемые в тексте изображения, хранятся в формате JSON в базе данных MySQL.
Все данные должны быть в кодировке UTF-8 без BOM для корректного отображения текста на русском языке.
Каждому уроку должна соответствовать отдельная HTML-страница

##Описание редактора
Добавление уроков должно быть похоже на добавление постов в блоге и осуществляться в редакторе Quill в админ-панели с возможностью выбора шрифта, цвета, и других параметров текста для содержимого уроков. При редактировании текста урока не должно быть необходимости использовать HTML-тэги. Все возможности редактирования должны быть обеспечены командами текстового редактора. Также нужна  возможность добавления таблиц, списков, ссылок, изображений в текст уроков. 
Форматы изображений: png, jpg, webp, svg. Для поддержки форматов изображений используй необходимые библиотеки. Нужна возможность загрузки файлов изображений через админку и сохранения их в файловой системе сервера, а также путей к файлам изображений в базе данных в записи соответствующего урока. Предварительный просмотр загруженных изображений Не нужен. Ограничения на размер загружаемых файлов: максимум 5 МБ для изображений. Сжатие/оптимизация изображений не требуется.
Подсветка кода в этом приложении не нужна.
В задачах должна быть возможность вставки фрагментов кода и изображений.
Текстовый редактор должен быть на странице администрирования (админ-панели, админке). 
Редактор должен сохранять данные в кодировке UTF-8 без BOM для корректного отображения текста на русском языке.


##Описание админ-панели
Добавление уроков должно быть похоже на добавление постов в блоге и осуществляться в редакторе Quill в админ-панели. 
Админ-панель должна быть на отдельной странице по адресу: <url сайта>/bod. Вход в админ панель должен быть с формой авторизации со следующими данными:
Логин: bodryakov.web
Пароль: Anna-140275
Логин и пароль должны быть зашиты в коде, хранить их в базе данных не нужно.
Вход в админ-панель должен быть только по URL. Кнопки входа на страницах сайта быть не должно.
Дополнительная защита от XSS/CSRF атак в админке не нужна.
Админ (он же редактор уроков) должен быть только один, добавление других админов не нужно.
В Админ-панели должно быть предусмотрено добавление и удаление разделов и уроков, а также редактирование заголовков разделов и уроков. Удаление, добавление и редактирование должно обеспечиваться соответствующими кнопками с соответствующими текстами и всплывающими подсказками.
При удалении, модальное окно подтверждения не нужно.
В момент сохранения названий разделов и уроков должна быть валидация:
1. Поле названия раздела или урока заполнено и название не повторяется
2. Поле slug заполнено и slug не повторяется. Slug может содержать только маленькие английские буквами и дефисы, пробелы запрещены.
3. Порядковый номер заполнен и не повторяется.
В Админ-панели должно быть предусмотрено создание страниц уроков и текстов уроков, тестов, задач с помощью полнофункционального редактора текста Quill. 
В интерфейсе создания тестов должны быть отдельные поля для вопроса, 4 вариантов ответов и checkbox для отметки правильного ответа.
Должна быть возможность добавлять изображения. Изображения могут добавляться как в текст теории, так и в текст задач. Изображения должны загружаться через стандартное диалоговое окно выбора файла. Реализовывать интерфейс drag and drop для загрузки файлов не нужно. Изображения должны сохраняться в файловой системе в каталоге соответствующего урока. При удалении урока изображения тоже должны удаляться из файловой системы.
При создании и редактировании урока должна быть кнопка “сохранить черновик”, которая сохраняет данные урока, но не публикует их, а также кнопка “опубликовать”, которая публикует урок в открытый доступ.
После редактирования информации в админ-панели все изменения должны сохранятся в соответствующих записях базы данных в формате JSON. 
Все данные должны быть в кодировке UTF-8 без BOM для корректного отображения текста на русском языке.

При выборе пользователем урока должна формироваться HTML-страница соответствующего урока на основе шаблона и данных из базы данных.
Изображения должны храниться в виде файлов в каталоге соответствующего урока.
Структурированный и форматированный текст урока должен хранится в формате JSON в базе данных.
Обработку URL осуществлять через PHP-роутинг.

##Правила для HTML-кода
Должен использоваться новейший (на 2026 год) семантический HTML. В HTML-коде должны быть классы названные в методологии БЭМ. Классы должны использоваться только для CSS стилизации. Для обращения к элементам из JavaScript должны использоваться data-атрибуты (только там где действительно нужен JavaScript)

##Правила для CSS-кода
Должен использоваться чистый, самый современный на 2026 год, CSS. 
Без поддержки старых браузеров. 
В качестве основного шрифта используй Roboto подключенный через Google Fonts.
Вместо Flex используй Grid. 

Должна быть адаптивность, со следующими значениями точек перелома:
Минимальная ширина: 320px
далее точки перелома:
< 600px
≥ 900px
≥ 1200px
≥ 1600px
≥ 1920px

Адаптивность должна быть сделана с использованием самых новейших возможностей CSS.
Используй следующую заготовку для media-запросов:

/* Базовые стили для мобильных устройств (от 320px) 
   Всё, что ниже 600px, описывается здесь без медиа-запроса */

/* Малые устройства (например, крупные смартфоны или маленькие планшеты) */
@media (width >= 600px) {
    /* Стили для экранов от 600px до 899px */
}

/* Средние устройства (планшеты в альбомной ориентации) */
@media (width >= 900px) {
    /* Стили для экранов от 900px до 1199px */
}

/* Ноутбуки и небольшие десктопы */
@media (width >= 1200px) {
    /* Стили для экранов от 1200px до 1599px */
}

/* Большие мониторы */
@media (width >= 1600px) {
    /* Стили для экранов от 1600px до 1919px */
}

/* Сверхширокие мониторы (Full HD и выше) */
@media (width >= 1920px) {
    /* Стили для экранов от 1920px и выше */
}

Максимальная ширина контента на экранах больше 1600 px должна быть ограничена 1600 px.

Вверху всех страниц сайта должна быть узкая (минимальной высоты) полоса header-а.
В этой полосе header-а, слева должен быть заголовок сайта: NewCSSLearn.
На ширинах >= 1200px должна также появляться в центре надпись: "новейший CSS c 2017 года" .
Должно быть переключением светлой и тёмной темы с кнопкой в header-е справа. 
Цвета предпочтительны серые, голубые, синие. Базовая тема светлая.
Промотка должна быть плавная. 

##Правила для JavaScript кода
В коде JavaScript, для поиска элементов через querySelectorAll() должны использоваться data-атрибуты, а не классы или ID. Классы должны использоваться только для CSS-стилизации. Соответствующие data-атрибуты должны быть в HTML-коде.
