# NewCSSLearn - Учебный курс по современному CSS

Полнофункциональное web-приложение для обучения студентов новейшим свойствам CSS с 2017 года.

## Возможности

- 📚 **Структурированный курс** - разделы и уроки с удобной навигацией
- 🎨 **Современный дизайн** - Material Design с темной/светлой темой
- ✍️ **Rich-редактор** - Quill.js для создания контента уроков
- 🧪 **Интерактивные тесты** - мгновенная проверка ответов
- 📝️ **Практические задачи** - задания для закрепления материала
- 🖼️ **Загрузка изображений** - поддержка форматов JPG, PNG, WebP, SVG
- 📱 **Адаптивный дизайн** - работает на всех устройствах
- 🌙 **Админ-панель** - полное управление контентом

## Технологический стек

- **Backend**: PHP 8.2 (без фреймворков)
- **База данных**: MySQL 8.0
- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **Редактор**: Quill.js (CDN)
- **Стилизация**: CSS Grid, Flexbox, CSS переменные
- **Контейнеризация**: Docker + Docker Compose

## База данных

Приложение использует две таблицы:

### sections (Разделы)
- `id` - уникальный идентификатор
- `title_ru` - название на русском
- `slug` - URL-псевдоним
- `section_order` - порядок сортировки

### lessons (Уроки)
- `id` - уникальный идентификатор
- `section_id` - ID раздела
- `title_ru` - название на русском
- `slug` - URL-псевдоним
- `lesson_order` - порядок в разделе
- `content` - контент в JSON
- `is_published` - флаг публикации

## Формат контента урока

```json
{
  "theory": "HTML контент теории",
  "tests": [
    {
      "question": "Текст вопроса",
      "answers": ["Ответ 1", "Ответ 2", "Ответ 3", "Ответ 4"],
      "correct": 2
    }
  ],
  "tasks": [
    {
      "title": "Название задачи",
      "description": "HTML контент условия"
    }
  ]
}
```

## Админ-панель

### Управление разделами
- Создание, редактирование, удаление разделов
- Валидация уникальности slug и порядка
- Просмотр уроков в каждом разделе

### Управление уроками
- Создание, редактирование, удаление уроков
- Rich-редактор Quill для контента
- Управление тестами и задачами
- Загрузка изображений
- Публикация/черновики

### Загрузка изображений
- Форматы: JPG, PNG, WebP, SVG
- Максимальный размер: 5 МБ
- Сохранение в `/uploads/lessons/{lesson_id}/images/`
- Автоматическое удаление при удалении урока

## Разработка

### Локальная разработка

1. **Запуск контейнеров в режиме разработки**
   ```bash
   docker-compose up
   ```

2. **Отслеживание изменений**
   - Файлы монтируются в реальном времени
   - Изменения сразу отражаются в контейнере

### Добавление нового функционала

1. **Создание миграции БД**
   - Добавьте SQL в `db-init.sql`
   - Пересоздайте контейнеры: `docker-compose down && docker-compose up`

2. **Создание новых страниц**
   - Добавьте маршрут в `index.php`
   - Создайте шаблон в `templates/`

3. **Добавление стилей**
   - Основные стили: `assets/css/main.css`
   - Админ-стили: `assets/css/admin.css`
   - Темы: `assets/css/themes.css`

## Производственные настройки

### Развертывание на хостинге

1. **Требования к хостингу**
   - PHP 8.2+
   - MySQL 8.0+
   - Apache с mod_rewrite
   - Поддержка .htaccess

2. **Настройка**
   - Загрузите файлы на хостинг
   - Настройте параметры подключения в `config.php`
   - Установите права на директорию `uploads/` (755)

3. **Настройка БД**
   - Импортируйте `db-init.sql`
   - Импортируйте `db-test-data.sql` (опционально)

## Траблшутинг

### Возможные проблемы

1. **Ошибка подключения к БД**
   - Проверьте статус контейнеров: `docker-compose ps`
   - Проверьте логи: `docker-compose logs db`

2. **Ошибка 404**
   - Проверьте настройки `.htaccess`
   - Убедитесь, что `mod_rewrite` включен

3. **Не загружаются изображения**
   - Проверьте права на директорию `uploads/`
   - Проверьте лимит размера файлов

### Полезные команды

```bash
# Перезапуск контейнеров
docker-compose restart

# Просмотр логов
docker-compose logs -f

# Очистка контейнеров
docker-compose down -v

# Вход в контейнер
docker-compose exec php bash
```
